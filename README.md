# Wildberries Product Parser

Парсер товаров **Wildberries**, который собирает результаты поиска и карточки товаров, а затем формирует **Excel-каталог**.

Проект состоит из двух краулеров:

1. **Producer** собирает товары из поиска WB.
2. **Consumer** загружает карточки товаров через `card.json`.

Результат объединяется и экспортируется в **Excel**.

---

## Результат работы

После выполнения пайплайна создаётся файл `products.xlsx`.

Он содержит:

- ссылку на товар
- артикул
- название
- цену
- описание
- ссылки на изображения
- характеристики
- название селлера
- ссылку на селлера
- размеры
- остатки
- рейтинг
- количество отзывов

## Требования

Перед запуском необходимо установить:

### 1. Google Chrome

Используется для получения **cookies Wildberries** через Playwright.

Без этого API WB может возвращать ошибки или блокировать запросы.

### 2. Docker Desktop

Используется для запуска **Redis** как очереди задач.

### 3. Python 3.10+

## Быстрый запуск

Из корня проекта выполните:

```bash
cp .env.example .env
python manage.py quickstart
```

Команда:

- создаст виртуальное окружение `.venv`
- установит Python-зависимости
- установит Playwright и Chromium
- запустит Redis через Docker
- обновит cookies WB
- запустит краулеры
- сконвертирует результат в Excel

После завершения появится файл `products.xlsx`.

## Архитектура проекта

Проект построен на очереди задач Redis.

```text
WB Search Spider
        |
        v
Redis Queue (nm_id)
        |
        v
WB Card Spider
        |
        v
JSONL
        |
        v
Excel
```

### Producer spider

Получает товары из поиска Wildberries и кладёт задачи в Redis.

Задача содержит:

```text
nm_id
source_item
```

### Consumer spider

Берёт задачи из Redis и загружает:

```text
https://basket-XX.wbbasket.ru/.../card.json
```

Результаты сохраняются в JSONL.

### Конвертер

Скрипт `src/scripts/jsonl_to_xlsx.py` преобразует JSONL в Excel.

## Особенности проекта

### 1. Работа напрямую с JSON API

Парсер не использует HTML-страницы.

Все данные берутся из внутренних API Wildberries:

```text
search API
card.json
```

Это делает парсинг:

- быстрее
- стабильнее
- менее подверженным изменениям вёрстки

### 2. Определение basket-сервера

Wildberries хранит товары на разных серверах. Сервер определяется по ID товара.

В проекте реализован быстрый алгоритм:

```text
nm_id -> vol -> basket
```

Через таблицу диапазонов определяется правильный URL без перебора серверов.

### 3. Очередь задач Redis

Используется схема:

```text
Producer -> Redis -> Consumer
```

Это позволяет:

- масштабировать парсинг
- разделять сбор поиска и карточек
- запускать несколько воркеров

### 4. Fallback-данные

Если `card.json` возвращает `404`, данные берутся из результата поиска (`source_item`).

Это позволяет избежать потери товаров.
